# 🛡️ WID-LB: WAF-Integrated Distributed Load Balancer
### L7 로드밸런서 연동형 분산 웹 방화벽 및 위협 시그니처 실시간 동기화 시스템 (PoC)

## 📌 프로젝트 개요
본 프로젝트는 단순 트래픽 분산에 머무르던 기존 L7 로드밸런서(LB)의 한계를 극복하고, 패킷의 페이로드를 심층 분석하는 **웹 방화벽(WAF, Web Application Firewall)** 엔진을 결합한 하이브리드 보안 아키텍처입니다.

신뢰할 수 없는 분산 환경(Zero Trust)에서 특정 워커 노드가 악성 패킷의 고유 패턴(시그니처)을 식별하면, 이 위협 정보를 **AES-GCM 알고리즘으로 안전하게 암호화하여 내부망(Redis)을 통해 모든 노드에 실시간 브로드캐스트**합니다. 이를 통해 단일 노드의 탐지 경험이 전체 시스템의 즉각적인 면역력으로 이어지는 유기적인 지능형 분산 방어망을 증명합니다.

## 🚀 핵심 아키텍처 및 기능 (Core Features)
1. **WAF 엔진 통합형 L7 로드밸런싱:**
   - 트래픽 분산(LB)과 보안 검사(WAF)를 단일 노드 내부 파이프라인으로 통합하여, 불필요한 네트워크 홉(Hop)을 줄이고 탐지 효율성을 극대화합니다.
2. **L7 트래픽 정규화 (Payload Normalization):**
   - 대소문자 변환, 공백/탭 제거, SQL 주석 절삭 등의 전처리 과정을 통해 공격자의 페이로드 변이(Mutation) 및 회피 공격을 원천 무력화합니다.
3. **시그니처 기반 지능형 탐지:**
   - IP 스푸핑(Spoofing) 등 외형적 정보 위조에 속지 않고, 정규화된 데이터 알맹이의 SHA-256 해시값을 추출하여 고유한 디지털 지문(시그니처)으로 식별·차단합니다.
4. **AES-GCM 위협 정보 동기화 (Zero Trust 통신):**
   - 분산 노드 간 위협 공유 시 발생할 수 있는 데이터 스니핑 및 변조를 방지하기 위해, 기밀성과 무결성(MAC Tag)을 동시에 보장하는 AES-GCM 암호화를 적용합니다.
5. **자동 해제 타임아웃 (TTL 기반 가용성 확보):**
   - 동일 IP 공유 환경(NAT) 등에서 정상 트래픽이 오탐되어 영구 차단되는 오동작을 방지하기 위해, Redis TTL을 활용하여 5분(300초) 후 차단 룰이 자동 해제되는 가용성 안전망을 구축했습니다.

## 🛠️ 기술 스택 (Tech Stack)
- **Infrastructure:** Docker, Docker Compose
- **Backend / Advanced LB:** Python (Flask), Requests
- **Database (Global Sync):** Redis
- **Security Logic:** Cryptography (AES-GCM), Hashlib (SHA-256)

## 💻 실행 및 검증 방법 (How to Run)
```bash
# 1. 인프라 및 통합형 로드밸런서 컨테이너 빌드 및 실행
docker-compose up -d --build

# 2. 페이로드 변이 공격 시뮬레이션 스크립트 실행 (IP 위조 및 구문 변이 테스트)
python3 attacker.py

# 3. 분산 방어망 실시간 동기화 상태 및 TTL 모니터링 (웹 브라우저 접속)
http://localhost:8081/admin/redis-status
```

## 💡 한계점 및 고도화 과제 (Improvements & Future Work)
실제 운영 환경(Production) 수준의 고도화를 위해 다음과 같은 추가 연구 및 개선 과제를 인지하고 반영할 계획입니다.

### 🕵️‍♂️ 1. 공격자 우회 및 시스템 무력화 대응 (Security Hardening)
- **다중 인코딩 난독화 대응:** URL/Hex/Base64 등이 중첩된 난독화 공격을 방어하기 위해 전처리 단계에 다중 디코딩 파이프라인 도입이 필요합니다.
- **자원 고갈(OOM) 공격 방어:** 대용량 페이로드나 초당 수만 건의 무작위 페이로드 폭격 시 메모리 과부하를 막기 위해, 페이로드 크기 제한(Length Limit) 및 L4 계층의 제한 로직 결합이 요구됩니다.
- **정교한 우회 패턴 분석:** HTTP 청크 분할 전송(Chunked Transfer) 등 패킷을 잘게 쪼개 보내는 공격에 대응하기 위한 스트리밍 버퍼링 조립 검사 로직이 필요합니다.

### 🙋‍♂️ 2. 서비스 가용성 및 오탐 방지 (Availability & UX)
- **정상 반복 트래픽 오탐 방지:** Heartbeat(ping)나 API 폴링 등 동일 데이터를 반복 전송하는 정상 요청의 오탐을 막기 위해, 단순 해시 일치 방식에서 정규표현식(Regex) 기반의 몽타주 패턴 동기화 구조로 확장이 필요합니다.
- **복호화 연산 병목 해결:** 요청마다 모든 차단 룰을 순회 복호화하는 구조를 개선하기 위해, 암호화된 룰셋을 주기적으로 로컬 메모리에 동기화 및 캐싱(Local Caching)하는 비동기 구조 아키텍처가 요구됩니다.
